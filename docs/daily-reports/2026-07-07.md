- pytest is the supported test runner; `manage.py test` requires
  `--settings=config.settings_test`.
- Exception messages are never persisted in sync run rows — class name only.

- **Phase 2 completion — Drive ingestion pipeline** (uncommitted at time of

  tests stay offline); `drive/export.py` (Docs→text/plain, Sheets→text/csv,

  the Phase 3 stub); admin-only, scoped-rate-limited, audited
  `POST /api/ingest/drive/sync/` that ignores request-body scope. Test suite

- **Phase 2 review-finding fixes** (this session, follow-up commit): metadata

## Decisions (Phase 2 completion)

- `creator_email` stays empty: Drive v3 has no creator field; recovering it
  needs the Revisions API (follow-up). Never fabricated from owner/modifier.
- Content lives in PostgreSQL (`BinaryField`) for the pilot scope; revisit
  with object storage if volume outgrows it.
- Extraction jobs are queued only after the sync transaction commits, so a
  rolled-back sync can never leave jobs pointing at missing rows.
- DRF throttle rate is env-configurable (`DRIVE_SYNC_THROTTLE_RATE`,
  default `10/hour`).
