  eight review findings:
  - Freshness aggregation now uses DB aggregates for backlog and evidence
    expiry and samples only the most recent completed runs
    (`FRESHNESS_RUN_SAMPLE_LIMIT`, default 20); the failure-streak reading
    caps at the sample size, which still drives a warning.
  - New daily `integrations.prune_completed_sync_runs` beat task deletes
    completed sync-run rows older than `SYNC_RUN_RETENTION_DAYS` (default 14),
    always keeping the latest successful run per target because delegated
    last-success is derived from run rows.

    `FRESHNESS_NEVER_SYNCED_GRACE_SECONDS` (defaults to the mode's sync
    interval); retrieval denies them regardless.
